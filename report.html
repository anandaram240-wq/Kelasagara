1. Introduction

KelasaGaara (ಕೆಲಸಗಾರ) is a hyper-local, bilingual (English and Kannada) service marketplace application developed for blue-collar daily-wage workers in Karnataka. The application organizes the informal job sector by enabling direct hiring, chat communication, and GPS map coordinates, eliminating middlemen and agency commissions.

2. Problem Statement

The blue-collar and daily-wage hiring market in India, specifically in Karnataka, is highly fragmented.

  • Workers (Masons, Plumbers, Electricians, Carpenters, Painters, etc.) have no digital presence and rely on physical assembly at labor stands.
  • Hirers face issues locating reliable service providers, negotiating wages, and communicating exact project addresses in rural or semi-urban taluks.
  • Existing platforms (like LinkedIn or Indeed) are built for formal white-collar jobs. Commercial home service directories (like Urban Company) charge high commissions (20% to 30%) and do not support small towns or rural areas.

3. Objectives

  • Zero-Commission Ecosystem: To facilitate direct worker-to-hirer connection.
  • Bilingual Interface: Support English and Kannada natively across all pages, categories, and inputs.
  • Bypass passwords: Utilize one-tap Google Sign-In to increase accessibility for users.
  • Interactive Location Pinning: Integrate a map canvas so hirers can pin exact work coordinates instead of typing long, ambiguous addresses.
  • Instant Synchronization: Establish low-latency WebSocket connections for real-time messaging, typing indicators, and immediate push alerts.

5. Detailed Technology Stack (Why We Used It)

  1. Frontend: React (SPA)
    • Why: Allows building a Single Page Application with reusable, component-based architecture. Fast DOM rendering keeps the app lightweight for mobile phones with slow internet connections.
  2. Build Tool: Vite
    • Why: Unlike Webpack, Vite uses native browser ES modules (ESM) to deliver fast Hot Module Replacement (HMR) and instantaneous build speeds.
  3. Database 1: Cloud Firestore (NoSQL Document Store)
    • Why: Designed for hierarchical document queries. We use it to filter workers by trade, district, and taluk, and perform quick searches on job listings.
  4. Database 2: Firebase Realtime Database (RTDB)
    • Why: Built on persistent WebSocket connections. It keeps database read/write latency below 100ms, which is critical for instant chat messages and real-time alerts.
  5. Authentication: Firebase Auth (Google Sign-In)
    • Why: Blue-collar workers often forget text passwords or usernames. Google Sign-In provides one-tap passwordless security.
  6. State Manager: Zustand
    • Why: A lightweight state store. It updates components dynamically without causing the unnecessary page re-renders associated with React Context.
  7. Maps Rendering: Leaflet JS (via CDN)
    • Why: An open-source mapping library. It bypasses expensive Google Maps JavaScript SDK keys, keeping project hosting completely free.
  8. Geocoding Engine: OpenStreetMap Nominatim API
    • Why: Converts latitude and longitude coordinates into actual street addresses (and vice versa) for free.
  9. Animations: Framer Motion
    • Why: Used to build smooth modal overlays, tab switches, and slide-in alerts.

6. Data Storage Details (Where & What)

We divide the data into two databases to optimize read/write costs and messaging speeds:

Database Data Type Collections / Paths Properties Stored
Cloud Firestore Persistent Structured Data users/{uid} User profiles, role, skills, daily rate, district, taluk, phone.
Cloud Firestore Queryable Jobs jobs/{jobId} Job title, description, skills required, district, taluk, pay, coordinates.
Cloud Firestore Booking Agreements bookings/{bookingId} Direct bookings, date, time, duration, status, payment terms, location coordinates.
Realtime DB Chat messages chats/{bookingId}/messages Message text, sender/receiver UIDs, timestamp, read status.
Realtime DB Presence Tracking presence/{userId} Online status (boolean), last active timestamp.
Realtime DB Real-time Alerts notifications/{userId} Live toast notifications for incoming requests.
